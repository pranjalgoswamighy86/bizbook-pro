⚠️ Emergency Database Backup

Download a complete copy of your entire BizBook Pro database. Works even if you can't log in normally.

When to use this: If you can't log in to the app, or if the regular backup option in Settings isn't working, this page will still let you download all your data using just your email and password.
What happens when I click Download?

Your email and password are verified against the BizBook Pro database. If correct, the server packages every table, every record (all companies, all sales, all purchases, all inventory — everything) into a single JSON file and downloads it to your computer.

The backup includes soft-deleted records too, so nothing is lost.

How long does it take?

Usually 5–30 seconds depending on how much data you have. For large databases (50,000+ records), it may take up to 2 minutes. Don't close this tab while the download is in progress.

How do I restore from this backup?

The downloaded JSON file contains step-by-step restoration instructions in its _metadata.instructions field. In short:

  1. Set up a new PostgreSQL database
  2. Deploy BizBook Pro pointing to that database
  3. Use the /api/db-admin endpoint with action restore to import this JSON
  4. Log in with your email + password — all data will be there
Prefer the command line?

You can also download a backup using curl:

curl -X POST https://YOUR-APP-URL.up.railway.app/api/backup/emergency \
  -H "Content-Type: application/json" \
  -d '{"email":"you@example.com","password":"yourpassword"}' \
  --output backup.json
Is this secure?

Yes — your password is verified against the database just like a normal login. We do not store your password, and the backup is only generated after successful authentication. The download is logged to the audit trail.

Even if your account has been deactivated or soft-deleted, you can still use this — the data is yours and you have the right to export it.

🔄 Restore from Backup

If your database was reset or you lost access, you can restore all your data from a previously-downloaded backup file. Enter your email + password (must match the backup file), then select the backup JSON file you downloaded earlier.

Note: The restore process only adds records that don't already exist. Existing records are NOT overwritten. This may take 1-5 minutes for large backups.
